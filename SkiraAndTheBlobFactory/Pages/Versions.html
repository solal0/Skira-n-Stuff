Blob Factory Versions & Updates

v1 - The beginning. >
Nothing much to say, website just started, you can import a file and that's it.
v3 - First upload. >
Users are now able to turn a 10Mb or smaller file into a blob url (e. g. blob:https://{random_string}.c.websim.com/your_file.txt). Those blob url only work locally.
v5 - Enhancement. >
Local blob url are shown below the preview and there's a small button next to it to copy the genereated link.
v8 - The beginning. >
Generated links are no longer local only, imported files are actually uploaded. (blob:https://{random_string}.c.websim.com/your_file.txt -> https://api.websim.com/blobs/your_file.txt).
v9 - Settings correctly implemented and updated. >
Settings popup now shows up when the gear button is clicked. New setting called "Big Files Handling" added, it can be checked to allow users to import a file of any size and when clicking Generating Link it cuts the file in multiple pieces of base64 and upload each part under a different blob, you can then copy all urls to clipboard.
v12 - Downloadable tickets. >
Once your big file has been fully uploaded you now have a button to download your file as a very small .txt file called a golden ticket. The .txt file contains all your file parts urls.
v13 - Ticket to original file tool is out !. >
I just released a python script/tool that allows ya'll to turn your golden ticket in the original file ! You can download this tool by clicking the new "Download Blob Compiler" and clicking Download ! mirror link button is... another way to download, just very much slower.
v18 - Lost ? Don't worry, infos are here. >
A I (info) button has been added, when clicked a popup is shown with some questions and answers. There's also a button who opens a discord invite to my discord server dedicated to the Factory.
v19 to v28 - Some small things I added. >
Added sample option in the info popup who provides 3 golden tickets to try by yourself. Added Accelerated Uploads option. When checked, big files parts are uploaded faster. Enhanced the settings popup to look more like the info one.
v29 - Added Terms of Service. >
When you open the website, the Terms of Service will now show up and ask you to agree and follow specific rules. Just letting you know that, if you do not agree and still use the website or do something that is agaisn't the Terms of Service can lead to legal actions depending what you did.

Not yet written...
v65 - This is another quick description. >
When Accelerated Uploads is checked, ETA (Estimated Time of Arrival) will be devided by 8 since the website uses 8 workers so the file upload is two times faster.
v66 - Number of workers changed to 24. >
After some testing I found that the most optimized number of workers was 24.
Enabling Accelerated Uploads will now generate 24 workers instead of 10.

Stats (calculated w/ a connection of 900/800Mbps) :
Upload VS Download
150Mb (15 parts) -> Upload: 16s / Download: 5s
958Mb (96 parts) -> Upload: 1m 07s / Download: 17s
Upload Optimization
[10 workers] 150Mb -> 45s
[10 workers] 100Gb -> ~2h-3h (calculated)
[24 workers] 150Mb -> 16s
[24 workers] 100Gb -> 1h43m (calculated)

Terms of Service changed to Terms of Use, the Terms are now an html hosted on github and shown on Skira's Blob Factory hosted on WebSim.

When uploading a 10Mb file, if the extension of it was unknown, the extension used in the url was .bin. This has just been fixed.
